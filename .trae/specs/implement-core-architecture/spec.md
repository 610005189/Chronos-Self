# Chronos-Self 核心架构实现 Spec

## Why
Chronos-Self 是一个创新的 AGI 架构，旨在解决当前大语言模型的核心缺陷：无时间连续性、无内源性驱动、无反思能力、无统一自我模型。通过实现该架构的核心组件，验证"自我即连续时间积分动态不变量"的核心假设。

## What Changes
- 实现双通道表征与交叉融合系统（语义意图流 + 逻辑物理流）
- 实现多时间尺度连续积分引擎（快变量 + 慢变量动力学）
- 实现内源性默认模式网络（混沌吸引子注入）
- 实现层级化自指与元认知调控（L0/L1/L2 三层结构）
- 实现反思机制与睡眠重放系统
- 实现工作记忆激活强度机制
- 实现完整的训练与验证框架
- **BREAKING** 这是一个全新的架构实现，不兼容现有Transformer范式

## Impact
- 受影响的领域：AGI研究、连续时间建模、有状态AI系统
- 核心文件系统：
  - 核心引擎：积分引擎、状态管理、动力学系统
  - 表征系统：双通道编码器、交叉注意力融合
  - 记忆系统：四层记忆架构、工作记忆管理
  - 训练系统：动力学对齐、长时序训练
  - 验证系统：涌现指标监测、P0-P2级验证

## ADDED Requirements

### Requirement: 双通道表征系统
系统应提供双通道输入表征能力，将外部输入分解为语义意图流和逻辑物理流。

#### Scenario: 语义意图编码
- **WHEN** 输入包含高层目标和情感倾向的文本
- **THEN** 语义编码器输出意图向量，捕获语用意图和情感倾向

#### Scenario: 物理逻辑编码
- **WHEN** 输入包含物体状态、空间位置、因果链条的结构化信息
- **THEN** 物理编码器输出逻辑向量，显式分离本体感觉和外部世界状态

#### Scenario: 交叉注意力融合
- **WHEN** 语义流和物理流需要融合
- **THEN** 通过双向交叉注意力生成 enriched 表征，意图查询物理，物理约束意图

### Requirement: 多时间尺度连续积分引擎
系统应实现基于Neural ODE的多时间尺度状态积分。

#### Scenario: 快变量实时演化
- **WHEN** 系统接收实时输入
- **THEN** 快变量以毫秒-秒级时间尺度连续演化，反映感知和情绪反应

#### Scenario: 慢变量稳定积分
- **WHEN** 快变量产生持续模式
- **THEN** 慢变量以小时-天级时间尺度缓慢更新，反映人格和长期目标

#### Scenario: 非对称耦合稳定性
- **WHEN** 快变量波动剧烈
- **THEN** 耦合系数自动降低，防止正反馈发散

### Requirement: 内源性默认模式网络
系统应在无外部输入时维持自持动力学，防止状态坍缩到寂灭不动点。

#### Scenario: 混沌注入激活
- **WHEN** 外部输入为零
- **THEN** 混沌吸引子产生持续扰动，维持系统活跃

#### Scenario: 自适应增益控制
- **WHEN** 核心子空间方差过高或过低
- **THEN** 自动调节混沌注入强度，维持在边缘混沌稳态

### Requirement: 层级化自指与元认知
系统应实现L0/L1/L2三层结构，提供元认知调控能力。

#### Scenario: L2元认知调控
- **WHEN** L1自我状态演化
- **THEN** L2通过固定投影提取高阶特征，输出元参数调控信号

#### Scenario: L2独立性验证
- **WHEN** 移除L2调控信号
- **THEN** L1应维持基本功能（维持率>0.4），证明自指截断有效

### Requirement: 反思与睡眠重放
系统应实现实时反思和离线睡眠重放机制。

#### Scenario: 实时反思
- **WHEN** 系统在线运行
- **THEN** 维护最近T步计算图，通过有限截断伴随法实时修正

#### Scenario: 睡眠重放
- **WHEN** 达到24模拟小时
- **THEN** 进入5分钟离线微调期，从关键帧重放并计算双重损失

### Requirement: 工作记忆机制
系统应实现激活强度瓶颈，模拟人类7±2组块容量限制。

#### Scenario: 组块容量限制
- **WHEN** 多个信息组块竞争注意力
- **THEN** 仅激活强度排名前N（N=7）的组块参与完整计算

#### Scenario: 历史信息保留
- **WHEN** 组块激活强度低于阈值
- **THEN** 信息被保留但不参与当前计算，支持快速恢复

### Requirement: 涌现验证系统
系统应提供动力学和行为学双维度涌现判定。

#### Scenario: 动力学序参量监测
- **WHEN** 系统运行
- **THEN** 实时计算状态自相关系数、最大李雅普诺夫指数、自预测误差

#### Scenario: 行为学指标判定
- **WHEN** 系统长期运行
- **THEN** 监测自发目标生成、跨场景知识迁移、干预后行为重组

### Requirement: P0级核心验证
系统应支持72小时无输入开环运行验证。

#### Scenario: 抗寂灭验证
- **WHEN** 系统无外部输入连续运行72小时
- **THEN** 慢变量基线漂移率低于阈值，λ_max保持稳定（0 < λ_max < 0.1）

#### Scenario: 动力学对齐验证
- **WHEN** 使用不同步长积分同一状态序列
- **THEN** 终点误差可控，半群性质成立
